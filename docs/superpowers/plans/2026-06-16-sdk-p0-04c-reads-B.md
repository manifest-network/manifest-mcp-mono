# SDK P0 — Plan 4c-reads-B: extract the 7 reads + full-set branded view types

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]` checkboxes.

**Goal:** Extract the **7 remaining P0a-spine reads** as ctx-shaped core free fns (`getLeasesByTenant`, `getLease`, `getLeaseByCustomDomain`, `getSKUs`, `getProviders`, `getBillingParams`, `getWithdrawableAmount`) from the lease-server inline bodies (+ net-new), each `fn(ctx: ReadCtx, input, opts?: CallOptions)` routed through `withReadSignal`. Return **core-owned branded view types** (`BrandedLease`/`BrandedSKU`/`BrandedProvider`) — branding the **full** scoped-id set on the read returns (adding `asAddress`/`asFqdn`). Convert the 4 remaining lease read handlers to thin callers. Additive on a green 4c-reads-A.

**Decisions locked (this session, research-confirmed):**
- **FULL-SET branding** (`idiom-research-4c-brand-completeness`, decisive on public-API stability): brand EVERY scoped id that appears on a read return — UUID trio + `Address` (`Lease.tenant`, `Provider.address` **and** `Provider.payoutAddress` — also a bech32 address; branding `address` but not `payoutAddress` would re-open the inconsistency the full-set decision closes) + `Fqdn` (`LeaseItem.customDomain`). Add `asAddress`/`asFqdn` (trust-casts, NO validation/normalize — chain reads are trusted; brands erase to `string` in JSON → wire/MCP outputSchema unchanged). Why now: branding a public read-return field later (`string → Fqdn`) is a forced major break for typed consumers (semver-ts); branding now is additive.
- **Brand-on-extraction via `as*`** (never `parse*` on chain reads). `getBillingParams` (no id fields) + `getWithdrawableAmount` (`Coin[]`) return UNBRANDED.
- **`getLease` consumers DEFERRED to P2.** B ships `getLease` as a net-new P0a surface fn (unit + cross-face + fixture); the 5 inline `billing.v1.lease` call sites in agent-core/fred (`troubleshoot.ts:82`, `manage-domain.ts:191`, `close-lease.ts:121`, `appStatus.ts:24`, `fetchActiveLease.ts:18`) stay positional with their bespoke error/null disambiguation until P2 ctx-ifies fred/agent-core positional DI as one unit.

**Architecture:** spec §5.4/§8/§9. Branded view types via **Omit-override** over the REAL manifestjs camelCase value types (full passthrough — carry every field, brand only the id-bearing ones), homed in `manifest-types.ts` beside the `DeployResult` precedent. The lease-server thin callers keep their camelCase MCP-output projection (now consuming already-branded ids, which re-stringify identically). `withReadSignal` self-acquires the rate-limit token, so each handler's explicit `acquireRateLimit` is DELETED.

**Tech Stack:** TypeScript ESM (`.js` imports), vitest 4 (+ `*.test-d.ts` typecheck), tsdown, `tsc --noEmit` lint (`noUnusedLocals`), Biome. Spec §5.4/§8/§9. Issue: ENG-309. Builds on 4a/4b/4c-0/4c-reads-A.
