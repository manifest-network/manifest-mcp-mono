# SDK P0 — Plan 4c-txs: ctx-ify the 3 on-chain txs + TxCallOptions

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]` checkboxes.

**Goal:** ctx-ify the 3 P0a-spine on-chain TX fns — `fundCredits`, `setItemCustomDomain`, `stopApp` — from positional to `fn(ctx: TxCtx, input, opts?: TxCallOptions)`, threading `TxCallOptions` (fee-wins / gasMultiplier / memo / signal) through the **extended `cosmosTx`** (broadcasting DIRECTLY — NOT `executeTx`, which is 4d). `setItemCustomDomain` moves to a discriminated-union input + branded `Fqdn`; the tx result id fields brand (`Address`/`LeaseUuid`/`Fqdn`). Convert the **8 call sites across 5 files** (lease `index.ts` ×3; fred `deployManifest.ts` ×1; agent-core ×4 — `close-lease.ts` ×1, `manage-domain.ts` ×1, `deploy-app.ts` ×2) + the high-churn tx mock tests. Completes 4c.

**Decisions locked (this session + the 4c-txs surface-map):**
- **DIRECT broadcast via the existing `cosmosTx`** (extended), NOT `executeTx` (no `executeTx` symbol exists; the per-signer mutex + multi-msg `executeTx` are 4d). Keeping the single `cosmosTx` chokepoint preserves the stringly `cosmos_tx` face byte-equivalence (§5.8) for free.
- **`TxCtx = Pick<CapabilityCtx,'chain'|'signer'|'logger'>`** (no query/fetch). **Sender resolution:** `sender = asAddress(await ctx.chain.getAddress())` — the wallet lives on `ctx.chain` (CosmosClientManager), NOT `ctx.signer`. **Do NOT call `requireAuthSigner(ctx)`** (it checks `ctx.signer`, which the call sites NEVER populate — agent-core's `manage-domain`/`close-lease` flows carry a `clientManager` but NO `walletProvider`/signer → it would throw `INVALID_CONFIG` on EVERY broadcast). The query-only guard ALREADY exists downstream: `cosmosTx → ctx.chain.getSigningClient()` throws `INVALID_CONFIG` when `rpcUrl`/`gasPrice` are absent. The Msg `sender` is set inside `cosmosTx` from the same `ctx.chain` wallet (consistent). 4c does NOT consume `ctx.signer` (reserved for 4d's per-signer mutex). See **OI-SENDER**.
- **Fee/memo channel** threaded as an OPT-IN extension at the `cosmosTx` boundary — do NOT widen the internal `TxOverrides`/`TxOptions`. **FEE-WINS** (bypass `buildGasFee`/simulate, pass `StdFee` straight to `signAndBroadcast` — the only path valid WITHOUT a configured gasPrice); **gasMultiplier** reuses the existing validation (simulate path); **fee + gasMultiplier = INVALID_CONFIG** (a NET-NEW guard); **memo** overrides `built.memo` at the join, fed to BOTH `buildGasFee`(simulate) AND `signAndBroadcast`(broadcast) — same bytes, else the gas estimate is wrong; **signal** bounds the CONFIRMATION await only.

**Architecture:** spec §5.5/§5.8. The signal can only stop you *awaiting* block inclusion — a submitted tx CANNOT be un-broadcast (cosmjs's `signAndBroadcast` COMBINES broadcast + the commit poll into one promise and accepts no `AbortSignal`, so there is no seam to race "only the confirmation"). So `withTxConfirmation` PRE-CHECKS `sig.aborted` before calling `broadcast` (an already-aborted call sends NOTHING) and otherwise races the whole `cosmosTx` promise vs the abort: an abort AFTER `broadcast` was called surfaces as `AbortError`/`TimeoutError` **while the in-flight tx may still commit** (the backgrounded `cosmosTx` runs to completion, its result discarded). This is the honest P0a semantic — the signal bounds the caller's WAIT, not the broadcast; a caller who aborts must treat the outcome as unknown (re-query the chain). **CONFIRMED (verified against `retry.ts`):** because the race lives OUTSIDE `cosmosTx`, the abort never enters `cosmosTx`'s internal `withRetry`, so it triggers NO re-broadcast (`retry.ts`'s transient set has no `abort`/DOMException pattern; the caller's `AbortSignal.timeout` `TimeoutError` is raised by the external race, not inside `cosmosTx`). `cosmosTx`'s own pre-existing transient-broadcast retry behavior is unchanged. A truly-cancellable confirmation-only path (`broadcastTxSync` + signal-bounded `pollForTx`) is deferred (4d/later).

**Tech Stack:** TypeScript ESM (`.js` imports), vitest 4 (+ `*.test-d.ts`), tsdown, `tsc --noEmit` lint (`noUnusedLocals`), Biome. Spec §5.5/§5.8. Issue: ENG-309. Builds on 4a/4b/4c-0/4c-reads.

```ts
import { type CallOptions, resolveCallSignal } from '../options.js';

/**
 * Bound the AWAIT of a broadcast+confirmation with the caller's effective AbortSignal (opts.signal + opts.timeout).
 * CRITICAL: a submitted tx CANNOT be un-broadcast (cosmjs's signAndBroadcast combines broadcast + the commit
 * poll and accepts no AbortSignal). This PRE-CHECKS `sig.aborted` BEFORE calling `broadcast` (an already-aborted
 * call sends NOTHING), then races the broadcast() promise vs abort. Once broadcast() is called the tx is sent;
 * an abort/timeout stops you AWAITING and surfaces AbortError/TimeoutError, but the tx MAY STILL COMMIT (the
 * losing broadcast() runs to completion in the background, its result discarded) — the signal bounds your wait,
 * NOT the broadcast; a caller who aborts must re-query the chain. Does NOT acquireRateLimit (cosmosTx does that).
 * A timeout rejects TimeoutError; a caller abort propagates the caller's reason.
 * NOTE: `broadcast` here is the WHOLE cosmosTx call (acquireRateLimit → getSigningClient → simulate → signAndBroadcast),
 * so an abort racing the early window (acquire/getSigningClient/simulate, BEFORE the wire send) ALSO surfaces
 * AbortError even though NO tx was sent. The caller cannot distinguish "aborted pre-send" from "aborted post-send,
 * still committing" from this seam — hence the conservative contract: on abort, treat the outcome as UNKNOWN and re-query.
 */
export async function withTxConfirmation<T>(
  broadcast: () => Promise<T>,
  opts?: CallOptions,
): Promise<T> {
  const signal = resolveCallSignal(opts);
  if (signal === undefined) return broadcast();
  if (signal.aborted) throw signal.reason; // BEFORE broadcast — no tx sent
  const p = broadcast();
  p.catch(() => {}); // swallow the losing branch's eventual rejection
  return new Promise<T>((resolve, reject) => {
    const onAbort = () => reject(signal.reason);
    signal.addEventListener('abort', onAbort, { once: true });
    p.then(resolve, reject).finally(() =>
      signal.removeEventListener('abort', onAbort),
    );
  });
}
```

  (Takes `CallOptions` — `TxCallOptions extends CallOptions`, so `signal`/`timeout` suffice; the fee/memo translation is separate, at the cosmosTx call. NOT barrel-exported.)

  | { leaseUuid: LeaseUuid; customDomain: Fqdn; serviceName?: string }

  readonly service_name: string;
  readonly custom_domain: Fqdn;

  readonly code: number;

  ctx: TxCtx,
